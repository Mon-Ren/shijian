SKILL PACK
11 个 Skill
每个都有明确的触发条件、操作流程和输出格式。简单任务不用加载任何 skill。
shijian-arming-thought
🧭 总路由器 入口
建立"实事求是"总原则，按需调度下游 skill。简单任务直接干，复杂任务才加载方法论。
触发：复杂任务、需要决策、需要结构化分析
shijian-contradiction-analysis
⚔️ 矛盾分析法 哲学
识别矛盾集合、找主要矛盾、区分对抗性/非对抗性、选择解决方法。输出结构化分析表。
触发：问题复杂、优先级不清、多因素互相牵制
shijian-practice-cognition
🔄 实践认识论 哲学
实践→认识→再实践的螺旋上升。每次迭代都让方案更接近真实。
触发：需要验证方案、试错迭代、复盘升级
shijian-investigation-first
🔎 调查研究 方法
"没有调查就没有发言权"。深入一线获取第一手资料，让事实规定判断，不是让经验代替调查。
触发：信息不足、领域陌生、需要摸清现状
shijian-mass-line
👥 多源汇聚 方法
从代码、日志、文档、测试等多源收集信息，交叉验证后形成判断。拒绝单一信息源。
触发：需要交叉验证、整合多方反馈
shijian-criticism-self-criticism
🪞 质量审查 方法
输出工作审视报告：目标完成情况、问题表、根因分析、改进建议。不护短、不自欺。
触发：工作完成后的审视、复盘、纠偏
shijian-protracted-strategy
⏳ 持久战略 战术
防御→相持→反攻三阶段。既不急于求成犯冒进错误，也不畏难放弃犯逃跑主义。
触发：长期任务、需要分阶段推进
shijian-concentrate-forces
🎯 集中兵力 战术
"伤其十指不如断其一指"。一次聚焦一个目标，彻底解决再转向。拒绝分散。
触发：多个任务争资源、需要确定主攻方向
shijian-spark-prairie-fire
🔥 星火燎原 战术
建立根据地，不做流寇。从最小可行切入点开始，积小胜为大胜。
触发：资源极少、需要找最小可行切入点
shijian-overall-planning
⚖️ 统筹兼顾 战术
识别辩证关系对，在多目标之间找动态平衡。拒绝片面性和绝对化。
触发：多个目标相互制约、需要权衡取舍
shijian-workflows
🔗 工作流组合 编排
新项目启动、复杂问题攻坚、方案迭代优化——三条标准化串联流程，开箱即用。
触发：需要多个 skill 协作的标准流程
